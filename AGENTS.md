# AGENTS.md

This repository is an AI hardware schematic design agent plus deterministic KiCad project compiler. Follow these rules for all future agent work.

## Product Scope

- Do not turn this into a keyboard PCB generator.
- V1 focuses on general hardware schematic design.
- V1 does not implement a complete PCB editor.
- V1 does not implement general autorouting.
- V1 does not integrate complex simulation.
- V1 is about requirement understanding, architecture generation, schematic IR, KiCad schematic generation, ERC checks, and report explanation.

## AI and Generator Boundary

- The AI must not directly write complex `.kicad_sch` or `.kicad_pcb` content.
- The AI may create or update structured design IR, schema, generator templates, rules, and planner actions.
- All KiCad files must be generated by deterministic generator code.
- Prefer changing IR, schema, generator, template, or rules before touching generated files.
- Generated workspace outputs are reproducible artifacts, not source-of-truth design intent.

## Tooling Boundary

- All external commands must go through `packages/tools`.
- `kicad-cli` must not be assumed to exist.
- Always detect `kicad-cli` availability and return a friendly error or mock report when unavailable.
- Do not shell out directly from UI or agent code.

## Code Standards

- Python must use type annotations.
- TypeScript must use strict types.
- Keep dependencies light unless there is a clear reason.
- Every new feature should update relevant docs.
- Preserve the V1 architecture: agent actions update IR, generator emits KiCad artifacts, tools wrap external commands.
